    12. Limitations and Bugs

    The Alembic Workbench is currently undergoing development and is thus susceptible to bugs. If while using the Alembic Workbench any undocumented features are observed, please contact David Day, day@mitre.org, Principal Investigator.

    Some of the limitations include:

    • Files created by the Alembic Workbench should be left in the directory in which they are created. For each file tagged in the interface, there is a central document description file that contains pointers to associated tag files, etc. If a tag file is moved, the pointers in the document description file will no longer be valid, unless manually updated to reflect the new location of the file.
    • Possibility for overlap between Alembic Workbench mouse bindings (see Table 1) and those specified by the user's window manager.
