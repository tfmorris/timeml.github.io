Data Preparation for TempEval

TempEval was accepted as task 15 for the SemEval-2007 workshop. Participation will hopefully get TimeML, and the tools we have written for it, out there. We thought this was a good idea in a moment of mental weakness. It is actually, but it involves more work than we bargained for.

For TempEval, we have defined three tasks:

TASK A: For a restricted set of event terms, identify temporal relations between events and all time expressions appearing in the same sentence. The restricted set of events contains the 59 stems that occur 20 times or more.
TASK B: For the samerestricted set of event terms, identify temporal relations between events and the Document Creation Time (DCT).
TASK C: Identify the temporal relations betweeen contiguous pairs of main verbs.

To reduce the complexity of the tasks, we adopted a restricted set of relation types: BEFORE, AFTER, OVERLAP and NON-RELATED. We have a task home page at http://timeml.org/tempeval/.

Part of the deal is to provide a dataset, which is built upon TimeBank 1.2. Here are some relevant TimeBank statistics:

documents 183
sentences 1967

As is clear in the table above, a fair amount of links need to be added. In fact, we even need to check all existing links because the quality of the TLINK tags is the main bottleneck in the current version of TimeBank and for an evaluation task we need to get the quality up.

So far, data preparation has gone though the following steps:

  1. Took all documents from TimeBank 1.2
  2. Ran closure over all documents
  3. Removed all tags that were not needed for tasks A, B and C (includes all SIGNAL, SLINK and ALINK tags as well as most TLINK tags).
  4. Mapped all relation types onto the restricted set used for TempEval: BEFORE, AFTER, OVERLAP, NON-RELATED.
  5. Added all TLINK tags that were needed for the tasks, but that were still missing, setting the relation type to UNKNOWN.

We will use Tango for adding the tlinks (all tasks) and a special tool for adding main event annotation.
